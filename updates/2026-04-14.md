## Troubleshooting Log

| Issue | Root cause | Fix |
|-------|-----------|-----|
| `vi.hoisted` pattern missing | `vi.fn()` declared before `vi.mock()` factory causes hoisting conflict | Use `vi.hoisted(() => ({ mockX: vi.fn() }))` in all mock factories |
| MyCatsPanel infinite fetch loop (2408 calls) | `vi.mock` returned a new object on every call → `user` ref changed every render → effect re-ran infinitely | Declare a stable `const user` inside the factory instead of the return expression |
| AuthContext tests failing after JWT decode | `login('fake-credential')` is not a valid JWT — `atob` throws | Replace test harness call with `buildFakeToken({ sub: 'u1' })` |
| `terraform apply` without confirmation | Ran `--auto-approve` without showing plan first | Always show plan output and ask before applying |

## CI/CD Pipeline Improvements

### 1. Unit test gate
- Added `test` job at the top of `deploy.yml` — backend/frontend typecheck and unit tests must pass before any deployment proceeds
- Previously CI and deploy ran independently, so a failing test would not block a deploy

### 2. UAT E2E tests
- Added `e2e-uat` job after `deploy-uat-frontend`
- Playwright hits the live UAT URL (`BASE_URL` environment secret) to verify the real deployment
- Prod deploy is blocked until E2E passes
- `playwright.config.ts`: added `BASE_URL` env var support — when set, the dev server is not started

### 3. Rollback workflow
- Added `rollback.yml` — triggered manually via `workflow_dispatch`
- Inputs: `environment` (uat/prod) and `git_sha` (target commit)
- Creates a `git revert` commit on main and pushes it — the deploy pipeline then runs automatically
- `main` HEAD always stays in sync with what is deployed

### 4. Missing handlers in deploy
- Fixed `deploy.yml`: `getMyCats` and `deletePhoto` were missing from all three environment handler lists

**Pipeline flow (push to main):**
```
test (gate) → deploy-uat-backend → deploy-uat-frontend → e2e-uat → deploy-prod (manual approval)
```
