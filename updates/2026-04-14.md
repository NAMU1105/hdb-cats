## S3 업로드 보안 강화

**파일:** `backend/src/lib/s3.ts`, `backend/src/handlers/getUploadUrl.ts`

### 1. Presigned URL ContentLength 서명
- `PutObjectCommand`에 `ContentLength` 포함 → S3가 선언된 크기와 다른 업로드 거부
- `thumbSizeBytes` 필드 추가: 추정값(20%) 대신 실제 thumb 크기를 클라이언트에서 전달

### 2. catId 존재 검증
- `catId` 제공 시 DynamoDB에서 해당 cat이 active인지 확인 후 presigned URL 발급
- 존재하지 않는 cat 폴더에 고아 파일 업로드 방지

### 3. API Gateway rate limit
- `POST /upload-url`: burst=5 / rate=2/s 추가 (기존 default burst=100 → 강화)

## 트러블슈팅

| 문제 | 원인 | 해결 |
|------|------|------|
| `vi.hoisted` 누락 | `vi.fn()` before `vi.mock()` factory → hoisting 충돌 | `vi.hoisted(() => ({ mockX: vi.fn() }))` 패턴 적용 |
| MyCatsPanel 무한 fetch (2408회) | `vi.mock` 매 호출마다 새 객체 반환 → user ref 변경 → effect 무한 실행 | factory 내부에 stable `const user` 선언 |
| AuthContext 테스트 실패 | `login('fake-credential')` 이 유효한 JWT 아님 → `atob` 실패 | 테스트 하네스를 `buildFakeToken({ sub: 'u1' })` 으로 교체 |
| Terraform apply 무단 실행 | plan 확인 없이 auto-approve 실행 | 다음부터 plan 보여주고 확인 후 진행 |
